# 2D gLSM reproduction of Yashin & Balazs 2007

Minimal NumPy/PyTorch reimplementation of the **two-dimensional gel lattice spring
model (gLSM)** from
> V. V. Yashin and A. C. Balazs,
> *Theoretical and computational modeling of self-oscillating polymer gels*,
> J. Chem. Phys. **126**, 124707 (2007), DOI 10.1063/1.2672951.

The paper combines:

1. **Modified 2-variable Oregonator** kinetics in the presence of a polymer
   diluent (eqs 4–7),
2. **Two-fluid model** linking solvent flux to polymer velocity field
   (eqs 8–16),
3. **Neo-Hookean rubber elasticity + Flory–Huggins mixing** free energy
   (eqs 19–25),
4. **Lattice spring discretisation** on a square lattice with bilinear
   quadrilateral elements (eqs 37–53), yielding explicit formulas for the
   nodal force (eq 48) and mobility (eq 53).

The reacting BZ gel is a chemomechanical transducer: the Ru(bpy)₃ catalyst,
covalently bound to the polymer network, alternately oxidises and reduces,
and this redox cycle modulates the polymer–solvent χ parameter (via the
coupling constant χ*), producing self-sustained swelling/deswelling waves.

## Scope of this reimplementation

Reproduces **Case I** boundary condition (small-IC + left edge pinned, as
in Figs 5(I) and 6 of the paper) on a 20×40 lattice with χ* = 0.105.
Goal: show the travelling oxidation wave in v and the accompanying
swelling wave in φ, matching the qualitative structure of Fig 6(B).

**Not implemented (yet):**
- Case II expansion conditions (Figs 9–11).
- Removal of boundary constraints (Fig 12, sample migration).
- The "drifting spiral tip" morphologies seen in the 20×40 Case II sample
  (Fig 11) — these require a larger, unconstrained domain.
- 3D scroll waves — those are the later 2008 PRE paper / the user's CUDA code.

## Layout

```
glsm/                      # core simulator library
  params.py                #   Tables I/II parameters + dx field (Δ=1 paper, Δ=0.5 Fortran)
  reaction.py              #   modified Oregonator F(u,v,φ), G(u,v,φ)   — eqs 6, 7
  mechanics.py             #   osmotic pressure, nodal force, mobility  — eqs 24, 25, 48, 53
  diffusion.py             #   interdiffusion flux J^(p) + J^(u)        — eqs 60, 61
  simulator.py             #   Fortran-strict RK4 + trapezoidal node update
  pino_losses.py           #   OregLoss for physics-informed FNO training
runs/                      # simulations / trainers
  generate_datasets.py     #   main dataset generator (rigid/deformable, multi-IC, --dx, --single-traj)
  make_warm_start_dx0p5.py #   build a clean-spiral warm-start .npz at Δ=0.5
  make_full_eval_from_split.py  #   combine train+test into CardiacEP-style Full_Set
  make_gif.py              #   frame-rendering helper used by --gif in generate_datasets
  PINO_Train.py            #   FNO / PINO trainer (Data-Only + Oregonator residual)
  run_case_I*.py / scan_fe.py / build_start_states.py   # historical single-case scripts
viz/                       # visualisation only, no simulation state
  plot_deform_gel.py       #   two-panel deformable-gel GIF (v on mesh + |δ|)
  make_error_gif.py        #   GT / Pred / Error GIF for a trained model
  plot_losses.py           #   training-loss curves (data-only vs PINO)
```

Gitignored (never tracked in this repo):

```
dataset/         # simulator output (.pt trajectories) — regenerate with runs/generate_datasets.py
results/         # training artefacts (checkpoints, logs, GIFs)
start_states/    # warm-start .npz snapshots
plots/           # figures generated by viz/ scripts
```

## Units and conventions

- Time in units of `T_0 = (k3·H·A)^-1 ≈ 1 s`.
- Length in units of `L_0 = sqrt(Du · T_0) ≈ 40 μm`, so one undeformed
  lattice element is `Δ = 1` (undeformed) with thickness `λ⊥·H_0`.
- Dimensionless fields `u = [HBrO2]/X_0`, `v = [Mox]/Z_0` per Tyson–Fife
  scaling.
- Nodes indexed as `(k, l)` with k spanning height, l spanning width.
  Element `m=(k,l)` has its lower-left corner at node `(k,l)`.

## Setup on a fresh machine

```bash
git clone https://github.com/lasite/bz-gel-pino.git
cd bz-gel-pino

# 1) Python env  (Python ≥ 3.11, CUDA 12.1 wheels assumed for torch 2.5.1).
python3.11 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

# 2) CardiacEP-PINOS  — vendored dependency, provides AP_neuralop_utils.
#    Clone it anywhere and point an env var at the checkout:
git clone <your-CardiacEP-PINOS-url> /path/to/CardiacEP-PINOS
export AP_UTILS_ROOT=/path/to/CardiacEP-PINOS
# (If AP_UTILS_ROOT is unset, runs/PINO_Train.py and viz/make_error_gif.py
#  fall back to the author's hard-coded path, which only exists on one
#  specific lab workstation.)

# 3) Regenerate data — nothing binary is tracked in git.
#    Rigid Δ=1 stable-spiral dataset (used for the FNO/PINO baseline):
python runs/generate_datasets.py --ic spiral --single-traj \
    --n-grid 101 --n-dumps 200 --dump-every 500 --dt-outer 0.001

#    Deformable Δ=0.5 spiral dataset from a warm-start:
python runs/make_warm_start_dx0p5.py            # → start_states/*.npz
python runs/generate_datasets.py --ic spiral --deform \
    --start-state start_states/spiral_dx0p5_*.npz \
    --n-grid 101 --n-dumps 360 --dump-every 500 --dt-outer 0.001 \
    --dx 0.5 --lambda0 100 --f 0.9
```

## Running

```bash
# sanity single-case runs (historical):
python runs/run_case_I_chi_star_0.py
python runs/run_case_I.py

# dataset visualisations:
python viz/plot_deform_gel.py -d dataset/spiral_deform/Train_*_5_inputsteps_5_outputsteps

# trainer (Data-Only baseline or PINO with Oregonator residual):
python runs/PINO_Train.py --phys-loss=False    # data-only
python runs/PINO_Train.py --phys-loss=True     # PINO

# post-training plots:
python viz/make_error_gif.py -r results/Results_... -d dataset/stable/Train_200_frames_...
python viz/plot_losses.py
```
